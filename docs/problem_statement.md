# Problem Statement

This is a summarized version of the problem statement document.

## Context

Small business rely on tools like

* Spreadsheets
* Messaging apps
* Emails
* Ad-hoc scripts

NOTE: These are business tools, think about more such tools.

Team should design and incrementally building a platform to support the core operational workflow of a small business.

## Objective

* Design and implement a software platform.
* Manage its internal operations in a structured, scalable, and maintainable way.
* The platform should be developed incrementally.
* With evolving requirements and explicit use of AI tools throughout the dev process.

## Scope

Define:

* Primary operational pain points.
* Key workflow to support.
* User roles involved.

## Funcational Expectations

* At least two distinct user roles (e.g. owner, employee, manager).
* Manage multiple interrelated workflows:
    * Orders
    * Inventory
    * Scheduling / assignments
    * Payments / invoices
    * Customer interations
* Well defined (tested) APIs for all core functionality.
* Automated tests for critical business logic.
* Design assuming future growth and change (agile dev cycle).
